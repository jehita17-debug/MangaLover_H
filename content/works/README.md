# 作品エントリの入れ方（けいた用・これだけ読めばOK）

1. `_TEMPLATE.md` をコピーして、英題のスラッグでファイル名を付ける。
   例: `Goodnight Punpun` → `goodnight-punpun.md`（小文字・スペースはハイフン）
2. **上の4〜5行だけ**埋める：

   | 項目 | 内容 |
   |---|---|
   | `title_en` | 英題（英語圏が検索する名前） |
   | `title_ja` | 日本語原題（正確に。差別化＆原題SEOの肝） |
   | `renta_url` | Renta! のアフィリリンク（バリューコマース経由） |
   | `mood_note` | **一言の角度メモ**＝「どの気分の夜に読む漫画か」。文体の核なのでここだけは丁寧に。 |
   | `status` | `draft` のまま置く |

3. コミットして push するだけ。GitHub Actions が動いて：
   - `draft` のエントリを拾う
   - 文体プロンプト（`prompts/review-prompt.md`）に沿ってレビュー本文・タグ・HOOK・比較を生成
   - エントリに書き戻して `status: published` に更新
   - サイトをビルドしてデプロイ

## status の値

- `draft` … 未生成。次回パイプラインで生成される。
- `published` … 生成済み・公開中。Claude API は呼ばれない（コスト節約）。
- `regenerate` … もう一度生成し直したい時にこれにする。

## やってはいけないこと

- `# AUTO-GENERATED BELOW` 以下の自動生成欄を手で書き換えない（次回上書きされる）。
- ネタバレを `mood_note` に書かない（本文に漏れる可能性）。深掘りは自動でネタバレ隔離欄に入る。
- 作品選定をAIに任せない。**何を載せるか＝けいたの審美眼**がこの案件の商品価値。
